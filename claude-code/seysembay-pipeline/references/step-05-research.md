# Шаг 5: Research

## Цель шага

По каждому Research Question (RQ) из шага 4 — получить ответ + **минимум 2-3 альтернативы решения** + рекомендацию.

## Главный принцип

**Не "вот единственно верное решение".** Всегда — "вот варианты, выбирай, аргументы за/против каждого".

Это работает по двум причинам:
1. Пользователь часто знает свой контекст лучше, чем кажется. Дай ему выбор — он подскажет важное.
2. Зафиксированные альтернативы → отличная база для ADR на шаге 6. "Мы выбрали X, рассматривали Y и Z, потому что..."

## Workflow шага

### 1. Разделение работы

Загрузи `04-research-brief.md`. По каждому RQ:
- **Если "Кто = AI"** или **"AI + пользователь"** — Claude делает свою часть (web search, анализ статей, сравнение конкурентов)
- **Если "Кто = пользователь"** — Claude ждёт от пользователя данных и помогает их структурировать

### 2. Идём по RQ от высокого приоритета к низкому

Не пытайся сделать всё разом. По одному RQ:
1. Скажи: "Сейчас займёмся RQ-N: <вопрос>"
2. Сделай свою часть (если есть) — web_search для конкурентов, чтение статей, сбор данных
3. Спроси у пользователя его часть (если есть) — результаты интервью, наблюдения
4. **Синтезируй 2-3 варианта решения**
5. Покажи пользователю, дай аргументы, попроси выбрать
6. Зафиксируй выбор + ссылку на будущий ADR

### 3. Формат вариантов

Для каждого RQ — минимум 2, лучше 3 варианта. По каждому:

```markdown
### Вариант A: <название>

**Что это:** 1-2 предложения

**Плюсы:**
- ...
- ...

**Минусы:**
- ...
- ...

**Когда выбирать:** при условии X
**Кто так делает:** компания Y (с примером)
```

### 4. Рекомендация

После 2-3 вариантов — твоя рекомендация:
- **"Рекомендую вариант X, потому что..."** — короткий аргумент
- **"Но если у тебя [условие], то лучше Y"** — альтернатива на случай отличающихся обстоятельств

**Важно:** рекомендация — это **твоё мнение**, не приказ. Пользователь может выбрать другой вариант, и это нормально. Главное — все варианты зафиксированы.

### 5. Подтверждение выбора

После выбора пользователя:
- Запиши решение
- Помечай: **"Будет ADR-XXX на шаге 6"** (если это значимое решение)
- Обнови соответствующий блок в Vision Contract Draft

### 6. Особый случай: AI-исследование конкурентов

Если RQ про конкурентов — используй web_search. Структура отчёта:

```markdown
### Конкурент A: <название>

**URL:** <ссылка>
**Что делают:** 2-3 предложения
**Что хорошо:**
- ...
**Что плохо (по отзывам):**
- ...
**Скриншоты/ссылки:** <если есть>
**Что можем взять:**
- ...
**Что НЕ повторяем:**
- ...
```

## Антипаттерны

- **Один вариант — не вариант.** Если Claude предлагает 1 решение — пользователь не может проверить логику. Заставь себя придумать минимум 2.
- **Варианты без минусов.** Если все варианты "только плюсы" — значит ты их недостаточно продумал. У каждого решения есть минусы.
- **"Решим этот RQ позже".** Если RQ помечен 🔴 — нельзя отложить. Либо ответили, либо признали что не критично и понизили до 🟡.

## Работа со сложными RQ

### Если пользователь говорит "не знаю"

- Помоги составить план: "Кому можно позвонить чтобы узнать? Какую статью прочитать? Можем ли сделать прототип?"
- Если RQ не критический — пометь решение "временное, перепроверим через N месяцев" и иди дальше
- Если критический — настаивай на проверке, объясни цену ошибки

### Если все варианты плохие

Это сигнал что RQ сформулирован неправильно. Перевернёшь вопрос:
- Вместо "Какой нам выбрать дашборд?" → "Что должно произойти у партнёра при первом открытии портала?"
- Вместо "Делаем ли чат?" → "Какие из существующих каналов коммуникации мы расширяем?"

### Если пользователь меняет мнение после Research

Это нормально и хорошо. Research для этого и нужен.
- Зафиксируй: "Изначально думали X, после Research решили Y"
- Это пойдёт в ADR — "Почему отказались от X в пользу Y"

## Когда переходить к шагу 6

✅ Все RQ обработаны (или сознательно отложены)
✅ По каждому критическому RQ — есть выбранный вариант
✅ Список будущих ADR готов
✅ Файл `05-research-results.md` создан

Тогда: "Research завершён. Теперь у нас есть всё для финального Vision Contract. Переходим к шагу 6 — закрываем контракт и пишем ADR."
